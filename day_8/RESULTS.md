Алгоритм быстрой сортировки (англ. QuickSort) — один из наиболее эффективных алгоритмов сортировки массивов. Его средняя сложность составляет $O(n \log n)$, хотя худший случай имеет сложность $O(n^2)$.

### Реализация быстрой сортировки на Python:

```python
def quicksort(arr):
    if len(arr) <= 1:  # Базовый случай: пустой список или список из одного элемента уже отсортирован
        return arr

    pivot = arr[len(arr) // 2]  # Выбираем опорный элемент (например, средний)
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    
    return quicksort(left) + middle + quicksort(right)  # Рекурсивная сортировка частей массива

# Пример использования:
arr = [9, 8, 7, 6, 5, 4, 3, 2, 1]
sorted_arr = quicksort(arr)
print(sorted_arr)
```

### Пояснения:
- **Опорный элемент** (`pivot`) выбираем посередине списка, чтобы избежать ситуаций, когда алгоритм работает неэффективно.
- Массив разбивается на три части:
  - Элементы меньше опорного элемента (`left`).
  - Элементы равные опорному элементу (`middle`).
  - Элементы больше опорного элемента (`right`).
- Затем рекурсивно сортируются обе части, кроме элементов, равных опорному.
- В конце объединяем результаты.

### Время работы:
- Средняя сложность: $O(n \log n)$
- Худшая сложность: $O(n^2)$ (возникает, если массив изначально отсортирован или почти отсортирован, и выбор опорного элемента неудачен).

### Преимущества:
- Эффективность средней сложности.
- Простая реализация.
- Быстрый доступ к произвольной позиции массива.

### Недостатки:
- Возможна плохая производительность в худших случаях.
- Требует дополнительной памяти для хранения промежуточных списков.

Для большинства практических случаев быстрая сортировка является оптимальным выбором среди простых алгоритмов сортировки.
